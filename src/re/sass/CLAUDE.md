## Build System — distcc (MUST reconstruct after reboot)

The build uses distcc via `/tmp/distcc-wrap/` — **this directory is in `/tmp` and is cleared on reboot**. After any reboot of x130e, recreate it:

```bash
ssh x130e
mkdir -p /tmp/distcc-wrap
cat > /tmp/distcc-wrap/cc << 'EOF'
#!/bin/sh
exec distcc /usr/bin/clang-19 "$@"
EOF
cat > /tmp/distcc-wrap/c++ << 'EOF'
#!/bin/sh
exec distcc /usr/bin/clang++-19 "$@"
EOF
chmod +x /tmp/distcc-wrap/cc /tmp/distcc-wrap/c++
```

## Active Bug Fixes in Mesa 26.0.3

### 1. GPU Hang: SX→CB deadlock (FIXED in r600_state_common.c)
- File: `src/gallium/drivers/r600/r600_state_common.c` — `r600_memory_barrier`
- Fix: emit `CS_PARTIAL_FLUSH` (drains all compute shader exports from SX→CB pipeline)
  BEFORE setting `R600_CONTEXT_FLUSH_AND_INV_CB`. Without this, SX and CB deadlock
  at 100% SX utilization, `radeon_fence_default_wait` never returns.

### 2. RAT readback: wrong output 0xFF292929 (PATCH APPLIED — awaiting test)
- File: `src/gallium/drivers/r600/evergreen_compute.c` — `compute_emit_cs`
- Root cause: `evergreen_emit_framebuffer_state` calls `r600_as_texture()` on pool_bo (a buffer,
  not a texture), producing garbage for CB_COLOR0_INFO and CB_COLOR0_CMASK.
- Fix: replaced `r600_emit_atom(rctx, &rctx->cb_state.atom)` with a direct
  `radeon_compute_set_context_reg_seq(cs, R_028C60_CB_COLOR0_BASE, 13)` that reads
  CB register values from `rctx->b.framebuffer.cbufs[0]` (r600_cb_surface, correctly
  populated by `evergreen_init_color_surface_rat`) and adds pool_bo relocations.
- Also: updated `r600_need_cs_space(rctx, 23, ...)` to account for 23 extra dwords.
- Status: compiled in build-terakan-distcc, awaiting install + test run.
